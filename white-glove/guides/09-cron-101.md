# Cron 101: Scheduling for n8n Users
*Replace n8n Schedule Triggers with native cron.*

## What is Cron?

Cron is a built-in scheduler on Mac/Linux. It runs commands automatically at set times. No extra software, no cloud service, no GUI.

**n8n equivalent:** Schedule Trigger node

## Why Switch from n8n?

| n8n Schedule Trigger | Cron |
|---------------------|------|
| Requires n8n running | Built into your Mac |
| Visual but limited | Text but powerful |
| Needs browser open | Runs in background |
| Cloud dependency | Local, always works |

## Cron Syntax

```
* * * * * command
│ │ │ │ │
│ │ │ │ └── Day of week (0-6, Sun=0)
│ │ │ └──── Month (1-12)
│ │ └────── Day of month (1-31)
│ └──────── Hour (0-23)
└────────── Minute (0-59)
```

## Common Patterns

| Schedule | Cron Expression |
|----------|-----------------|
| Every minute | `* * * * *` |
| Every hour | `0 * * * *` |
| Every day at 9am | `0 9 * * *` |
| Every Monday at 9am | `0 9 * * 1` |
| Weekdays at 9am | `0 9 * * 1-5` |
| First of month at midnight | `0 0 1 * *` |

## Setting Up Cron

### Step 1: Open Cron Editor
```bash
crontab -e
```
First time? It will ask which editor. Type `nano` and press Enter.

### Step 2: Add Your Job
Add a line like:
```bash
0 9 * * * /usr/local/bin/claude skill run daily-report
```
This runs `/daily-report` skill every day at 9am.

### Step 3: Save and Exit
- Press `Ctrl+O` to save
- Press `Enter` to confirm
- Press `Ctrl+X` to exit

### Step 4: Verify
```bash
crontab -l
```
Shows your scheduled jobs.

## Real Examples

### Daily Report at 9am
**n8n:** Schedule Trigger → HTTP Request → Email
**Cron:**
```bash
0 9 * * * cd ~/projects/myapp && /usr/local/bin/claude -p "Generate daily report and save to reports/"
```

### Weekly Backup on Sunday
**n8n:** Schedule Trigger → Code Node → Upload
**Cron:**
```bash
0 2 * * 0 cd ~/projects && tar -czf backup-$(date +%Y%m%d).tar.gz . && mv *.tar.gz ~/backups/
```

### Check API Every Hour
**n8n:** Schedule Trigger → HTTP → IF → Slack
**Cron:**
```bash
0 * * * * curl -s https://api.example.com/health || echo "API down" >> ~/logs/alerts.log
```

## Migrating n8n Schedules

### Step 1: List Your n8n Triggers
Open n8n, find all workflows with Schedule Trigger. Note:
- What time they run
- What they do
- What output they produce

### Step 2: Convert to Commands
For each workflow:
1. What would you ask Claude Code to do?
2. Can it be a skill? (reusable = yes)
3. Write the cron line

### Step 3: Test Before Scheduling
Run the command manually first:
```bash
cd ~/myproject && claude -p "Run my daily task"
```
Works? Add to cron.

## Using Claude Code Skills with Cron

Skills are perfect for cron because they're repeatable.

**Create a skill once:**
```bash
cd ~/myproject
claude
# "Create a skill called daily-summary that..."
```

**Schedule it:**
```bash
0 8 * * * cd ~/myproject && /usr/local/bin/claude skill run daily-summary
```

## Logging Output

Cron runs silently. Capture output:
```bash
0 9 * * * cd ~/myproject && claude -p "task" >> ~/logs/task.log 2>&1
```

The `>> ~/logs/task.log 2>&1` part:
- `>>` appends to file
- `2>&1` includes errors

## Common Issues

**Command not found**
Use full paths:
```bash
# Wrong
0 9 * * * claude skill run report

# Right
0 9 * * * /usr/local/bin/claude skill run report
```

Find your path: `which claude`

**Wrong directory**
Always `cd` first:
```bash
0 9 * * * cd /Users/you/projects/myapp && /usr/local/bin/claude -p "task"
```

**Not running**
Check cron is enabled:
```bash
# Mac - grant Terminal full disk access in System Preferences
# System Preferences → Security & Privacy → Full Disk Access → Terminal
```
