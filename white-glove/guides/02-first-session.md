# Your First 30 Minutes with Claude Code
*You've installed Claude Code. Now what?*

## Starting Claude Code
1. Open Terminal
2. Navigate to project: `cd ~/Desktop/my-project`
3. Type `claude` and press Enter

## Your First Conversation

**Explore your project:**
```
What is this project? Give me a quick overview.
```

**Ask for help:**
```
I want to add a dark mode toggle. How should I approach this?
```

**Fix something:**
```
The login button isn't working. Help me debug it.
```

**Learn something:**
```
Explain how the authentication flow works in this codebase.
```

## Key Commands

| Command | What It Does |
|---------|--------------|
| `/help` | Show all commands |
| `/clear` | Start fresh |
| `/compact` | Summarize and continue |
| `Ctrl + C` | Stop or exit |

## How Claude Code Works

1. **You ask** → Claude reads relevant files
2. **Claude proposes** → Shows you the plan
3. **You approve** → Claude makes changes
4. **You verify** → Check the results

**Important:** Claude asks permission before modifying files. Read proposed changes before approving.

## Tips for Better Results

**Be specific:**
- Bad: "Fix the bug"
- Good: "The submit button on /login isn't sending form data"

**Give context:**
- Bad: "Add a feature"
- Good: "Add a 'Remember me' checkbox that persists session for 30 days"

**Point to files:**
- Bad: "Update the styles"
- Good: "Update src/components/Button.css to make hover state visible"

## Common First Tasks

**Set up CLAUDE.md:**
```
Create a CLAUDE.md file for this project with my preferences.
```

**Understand the codebase:**
```
Give me a map of this codebase. Main files and how they connect.
```

**Fix a small bug:**
```
There's a typo in the footer. "Copywrite" should be "Copyright". Fix it.
```

**Add a simple feature:**
```
Add a "Back to top" button that appears when scrolling down.
```

## What's Next?

1. Create your CLAUDE.md
2. Try a real task from your to-do list
3. Learn slash commands (`/help`)
4. Practice daily - 1 task per day
